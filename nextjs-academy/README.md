# Academy — Next.js

Next.js (App Router) build of the Academy workshop site. This was migrated from
the original HTML design **with no visual or UX changes** — same layout, styles,
copy, animations, and interactions.

## Run it

```bash
npm install
npm run dev
```

Open http://localhost:3000.

## Routes

| Route | Page |
|---|---|
| `/` | Workshop landing (public) |
| `/signin` | Sign in |
| `/onboarding` | Onboarding wizard |
| `/dashboard` | Dashboard |
| `/workshop` | Workshop (logged in) |

## How it's structured

```
app/
  layout.jsx          # <html>/<body>, Geist font, global CSS
  globals.css         # resets, button system, media queries, keyframes,
                      #   and hover/focus/active rules (all ported verbatim)
  page.jsx            # "/"           (imports lib/logic/home)
  workshop/page.jsx   # "/workshop"
  dashboard/page.jsx  # "/dashboard"
  signin/page.jsx     # "/signin"
  onboarding/page.jsx # "/onboarding"
lib/
  dc.js               # css() style helper + useDcLogic() adapter + DCLogic base
  logic/*.js          # the page interaction logic (state, effects, handlers)
public/assets/        # images
```

### Design notes for developers

- **Styling is 1:1 with the original.** Inline styles are applied through the
  `css()` helper (`lib/dc.js`), which parses the original CSS declaration strings
  into React style objects — so the values match the source exactly, including
  CSS custom properties (the theme tokens live on each page's root element).
  Anything that can't be inline (button system, `@media`, `@keyframes`,
  `:hover`/`:focus`/`:active` states) lives in `app/globals.css`.
- **Interaction logic** for each page is a small class in `lib/logic/`. It runs
  through `useDcLogic()` (a thin React adapter). State, `componentDidMount`
  effects (IntersectionObserver scroll-reveals, the `/workshop` countdown to
  Jul 31, the `/signin` benefits carousel, profile-dropdown/avatar upload, PDF
  modal, onboarding steps) are all preserved.
- **Navigation** uses plain `<a href="/route">`. Swap to `next/link`'s `<Link>`
  if you want client-side transitions — the routes already match.
- **Hover states** were converted to attribute-scoped CSS rules
  (`[data-h="…"]:hover { … }`) so they win over inline styles; you'll see
  `data-h` / `data-hf` / `data-hac` attributes on elements that had hover/focus/
  active styling.

If you'd rather hand-author cleaner components, the original design files and a
full migration spec are in the sibling `design_handoff_academy_nextjs/` bundle.
